Fundamentos de Vue.js
Arquitectura, componentes y ciclo de vida
Estructura componentizada
La aplicación se divide en piezas reutilizables e independientes (componentes) que encapsulan su propia lógica, plantilla y estilos.
DOM Virtual y componentes
Vue mantiene una representación ligera del DOM en memoria; al detectar cambios sólo actualiza los nodos necesarios, mejorando el rendimiento.
Aplicaciones reactivas
Cuando los datos cambian, la vista se actualiza sola. Vue observa el estado y re-renderiza sólo las partes afectadas.
Componentes — Concepto
Bloque de construcción que combina template + script + style en un archivo .vue. Puede recibir props y emitir eventos.
Utilización de componentes
Los componentes se registran globalmente o localmente, se importan con ES Modules y se usan como etiquetas HTML personalizadas.
Concepto y creación de SPAs
Single Page Application: carga una sola página HTML y actualiza el contenido dinámicamente usando Vue Router, sin recargas completas.
Ventajas de SPAs
Navegación fluida sin recarga, mejor UX, API centralizada, rendimiento optimizado y separación clara entre frontend y backend.
App como árbol de componentes
App.vue es el componente raíz que orquesta el árbol completo; cada componente puede contener hijos formando una jerarquía.

Vista — Template & Datos
Sintaxis, directivas y comunicación entre componentes
Template como parte del componente
La sección <template> dentro del SFC define el HTML del componente, directamente junto al script y los estilos.
Usando un template externo
Es posible separar la plantilla en un archivo HTML independiente y referenciarla, aunque los SFC son la práctica recomendada.
Sintaxis del template
Interpolación {{ expr }} para texto, v-bind (o :) para atributos dinámicos, expresiones JS válidas.
Expresiones
El template acepta expresiones JS simples: operaciones, ternarios y llamadas a métodos. No admite sentencias completas (if, for literales).
Directivas
Atributos especiales prefijados con v- que aplican comportamiento reactivo: v-if, v-for, v-model, v-on, v-show…
Manejo de datos entre componentes
Props (padre→hijo) y emit (hijo→padre). Para estados globales se usan Pinia o Vuex; para vecinos, un bus o provide/inject.
Propiedades computadas vs observadas
computed: valor derivado con caché automático. watch: reacciona a cambios con efectos secundarios (peticiones, logs).
Validación y valores por defecto
Las props aceptan type, required, default y validator, definiendo un contrato claro entre padre e hijo.
Scopes
Cada componente posee su propio scope de datos aislado. Los datos no son accesibles directamente desde componentes hermanos.
Acceso a root y parent
$root apunta a la instancia raíz y $parent al padre directo. Se recomienda usarlos con moderación para evitar acoplamiento.
<!-- v-if: renderizado condicional — el nodo no existe en el DOM si es false -->
<p v-if="isLoggedIn">Bienvenido, {{ username }}</p>

<!-- v-for: iteración — genera un elemento por cada ítem del array -->
<li v-for="item in items" :key="item.id">{{ item.name }}</li>

<!-- v-model: enlace bidireccional entre input y propiedad reactiva -->
<input v-model="email" type="email" />

<!-- computed: valor derivado con caché (solo se recalcula si cambia nombre) -->
const fullName = computed(() => `${firstName.value} ${lastName.value}`)

Comportamientos
Métodos, eventos, formularios y persistencia
Métodos
Funciones definidas en methods (Options API) o directamente en setup() que encapsulan la lógica de negocio del componente.
Eventos de teclado y mouse
Con @keydown.enter, @click, @mouseover Vue escucha eventos nativos. Se pueden encadenar modificadores (.prevent, .stop).
Mixins
Objeto reutilizable que aporta lógica compartida (data, methods, hooks) a varios componentes. En Vue 3 se prefieren los Composables.
Emisión y escucha de eventos
El hijo notifica al padre con emit('evento', payload); el padre escucha con @evento="handler". Flujo de datos unidireccional y explícito.
Formularios y v-model
v-model enlaza bidireccionalmente inputs, checkboxes, selects y textareas a propiedades reactivas, simplificando la captura de datos.
Campos dependientes
Usando v-if / v-show condicionados a propiedades del formulario se muestra u oculta cada campo según el estado actual.
Validación en tiempo real
Watchers o computed detectan errores mientras el usuario escribe. Librerías como VeeValidate o Zod estructuran las reglas de forma escalable.
Persistencia de datos
El estado se puede guardar en localStorage / sessionStorage o sincronizar con un API REST/GraphQL para sobrevivir recargas.
Reconstrucción del estado
Al montar el componente (onMounted) se leen los datos guardados y se restaura el formulario al estado previo del usuario.
// Método llamado desde el template con @submit.prevent
function handleSubmit() {
  if (!email.value) { errors.value.email = 'Requerido'; return; }
  localStorage.setItem('form', JSON.stringify({ email: email.value }))
}

// Escuchar evento personalizado del hijo en el padre
// Hijo:   emit('user-saved', user)
// Padre:  <UserForm @user-saved="onUserSaved" />
function onUserSaved(user) { users.value.push(user) }

Animaciones & Utilidades
Estilos dinámicos, transiciones, AXIOS y herramientas
Estilos como atributos
:style="{ color: textColor, fontSize: size + 'px' }" aplica CSS inline dinámico usando un objeto o array de objetos.
Clases CSS dinámicas
:class="{ active: isActive, error: hasError }" activa o desactiva clases según booleanos reactivos. Compatible con clases estáticas.
Transiciones
El componente <Transition> envuelve elementos e inyecta clases CSS de entrada/salida (v-enter-from, v-leave-to) para animar con CSS o JS hooks.
Renderizado condicional
v-if elimina/crea el nodo del DOM; v-show sólo alterna display:none. Usar v-show para toggles frecuentes.
Incorporación de librerías externas
Se instalan via npm y se importan en el componente o se registran globalmente en main.js. Ej: Chart.js, VueUse, Day.js.
Envío y recepción con AXIOS
AXIOS es una librería HTTP basada en promesas. Permite GET, POST, PUT, DELETE, interceptores globales y cancelación de peticiones.
Vue Devtools
Extensión de Chrome/Firefox que permite inspeccionar el árbol de componentes, el estado reactivo, los eventos emitidos y las rutas en tiempo real.
Configuraciones para debug
En modo development, Vue activa mensajes de advertencia detallados. Vite/Webpack generan source maps para rastrear errores hasta el código fuente.
// Petición GET con AXIOS — se ejecuta al montar el componente
import axios from 'axios'

onMounted(async () => {
  const { data } = await axios.get('/api/users')
  users.value = data
})

<!-- Transition anima la aparición/desaparición del listado -->
<Transition name="fade">
  <ul v-if="users.length">
    <li v-for="u in users" :key="u.id">{{ u.name }}</li>
  </ul>
</Transition>

/* CSS para la transición fade */
.fade-enter-active, .fade-leave-active { transition: opacity .4s ease }
.fade-enter-from,  .fade-leave-to     { opacity: 0 }
Consejo: Instala la extensión Vue Devtools desde la Chrome Web Store y activa el panel de Vue en DevTools (F12) para depurar componentes, estado y eventos en tiempo real.
