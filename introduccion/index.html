01 — Estructura

Aplicación Componentizada

En Vue, toda la interfaz se construye con componentes: bloques autónomos que encapsulan su propio template, lógica y estilos. La app raíz es un árbol de componentes anidados.

🌳 App Root

App como árbol de componentes

La instancia raíz <App /> monta toda la aplicación. Cada componente hijo puede tener sus propios hijos, formando un árbol. Esto facilita la separación de responsabilidades y el mantenimiento.

📄 SFC

Single File Component (.vue)

Un .vue agrupa en un único archivo: <template> (HTML), <script> (lógica JS) y <style> (CSS). Esto promueve la cohesión y facilita el razonamiento sobre el componente.

🔗 Comunicación

Props & Eventos

Los componentes se comunican con props (datos del padre al hijo) y emit (eventos del hijo al padre). Esta unidireccionalidad hace el flujo de datos predecible.

MiComponente.vue 
<!-- TEMPLATE: Define la estructura HTML del componente -->
<template>
  <div class="tarjeta">
    <!-- Interpolación reactiva: muestra el valor de 'titulo' -->
    <h2>{{ titulo }}</h2>

    <!-- v-if: renderizado condicional -->
    <p v-if="visible">{{ descripcion }}</p>

    <!-- @click: manejo de eventos, llama a 'alternar()' -->
    <button @click="alternar">Mostrar/Ocultar</button>
  </div>
</template>

<!-- SCRIPT: Lógica del componente con Composition API -->
<script setup>
import { ref } from 'vue'

// defineProps: declara las propiedades que recibe del padre
const props = defineProps({
  titulo: String,       // prop 'titulo' de tipo String
  descripcion: String  // prop 'descripcion' de tipo String
})

// ref(): crea una referencia reactiva (estado interno del componente)
const visible = ref(false)

// Función que modifica el estado reactivo
function alternar() {
  visible.value = !visible.value  // el DOM se actualiza automáticamente
}
</script>

<!-- STYLE SCOPED: CSS que solo aplica a ESTE componente -->
<style scoped>
.tarjeta { padding: 1rem; border-radius: 8px; }
</style>
02 — Rendimiento

DOM Virtual & Componentes

El Virtual DOM es una representación en memoria del árbol real del DOM. Vue calcula las diferencias (diffing) y aplica solo los cambios mínimos necesarios, haciendo las actualizaciones eficientes.

🧠 VDOM

¿Por qué Virtual DOM?

Manipular el DOM real es costoso. El VDOM actúa como buffer: Vue compara el árbol virtual anterior con el nuevo, detecta diferencias mínimas y las aplica en batch al DOM real.

🔍 Diffing

Algoritmo de reconciliación

Vue 3 usa un algoritmo de diffing optimizado con key. Al identificar nodos únicos, puede reutilizar elementos DOM existentes en lugar de recrearlos desde cero.

⚙️ Renderer

Renderizador personalizable

Vue 3 separa el núcleo reactivo del renderizador. Esto permite usar Vue para render en Canvas, WebGL, nativo (Vue Native) o cualquier entorno distinto al navegador.

Flujo de actualización Vue 
/*
  FLUJO DEL VIRTUAL DOM EN VUE 3:

  [Estado cambia]
       │
       ▼
  [Función de render → genera nuevo VNode Tree]
       │
       ▼
  [Diff Algorithm → compara VNode anterior vs nuevo]
       │
       ▼
  [Patch → aplica SOLO las diferencias al DOM real]
       │
       ▼
  [DOM actualizado eficientemente] ✅
*/

// Ejemplo: VNode (nodo del Virtual DOM)
const vnode = {
  type: 'div',           // tipo de elemento
  props: {
    class: 'tarjeta',    // atributos/propiedades
    id: 'main'
  },
  children: [            // nodos hijos
    { type: 'h1', children: 'Hola Vue' },
    { type: 'p',  children: 'DOM Virtual' }
  ]
}

// Vue convierte el template en una función render:
function render() {
  return h('div', { class: 'tarjeta' }, [
    h('h1', null, estado.titulo),  // reactivo al estado
    h('p',  null, estado.texto)
  ])
}
03 — Reactividad

Aplicaciones Reactivas del Mundo Real

El sistema reactivo de Vue 3 usa Proxy para interceptar accesos y mutaciones de datos. Cuando un dato cambia, Vue sabe exactamente qué partes del DOM actualizar.

Estado reactivo: contador
0
Datos derivados reactivos
contador.value: 0
doble (computed): 0
cuadrado (computed): 0
es par (computed): Sí
último cambio: —
reactividad.js — Composition API Vue 3 
import { ref, reactive, computed, watch } from 'vue'

// ref(): envuelve primitivos en un objeto reactivo
// Acceso mediante .value en el script
const contador = ref(0)

// reactive(): hace reactivo un objeto completo (sin .value)
const usuario = reactive({
  nombre: 'Ana',
  edad: 25,
  activo: true
})

// computed(): valores derivados que se recalculan automáticamente
// Solo se ejecuta si sus dependencias cambian (memoizado)
const doble    = computed(() => contador.value * 2)
const cuadrado = computed(() => contador.value ** 2)
const esPar    = computed(() => contador.value % 2 === 0)

// watch(): reacciona a cambios con lógica personalizada
// Útil para llamadas API, guardado en localStorage, etc.
watch(contador, (nuevoValor, valorAnterior) => {
  console.log(`Cambió de ${valorAnterior} → ${nuevoValor}`)
  // Aquí iría lógica como fetch() o guardar en BD
})

// watchEffect(): se ejecuta inmediatamente y rastrea
// automáticamente sus dependencias reactivas
watchEffect(() => {
  document.title = `Contador: ${contador.value}`
})
04 — Arquitectura

Concepto y Creación de SPAs

Una Single Page Application carga una única página HTML y reemplaza el contenido dinámicamente al navegar. Vue Router gestiona las rutas del lado del cliente, sin recargar el navegador.

http://miapp.com/#/inicio

🏠 Inicio — Vista Principal

Esta es la vista de inicio de la SPA. No se recargó el navegador al navegar aquí. Vue Router interceptó el cambio de ruta y reemplazó este componente dinámicamente. El HTML base (index.html) nunca cambia.

👤 Perfil de Usuario

Cada "página" es en realidad un componente Vue que se monta y desmonta. Vue Router escucha el hash/history de la URL y decide qué componente renderizar en el elemento <RouterView />.

⚙️ Configuración

Las rutas se definen en un objeto de configuración: { path: '/config', component: Config }. Supports rutas anidadas, parámetros dinámicos (/user/:id), guards de navegación y carga lazy.

ℹ️ Acerca de esta SPA

Vue Router usa la History API del navegador para cambiar la URL sin recargar. En modo hash (#/ruta) funciona sin configuración de servidor. En modo HTML5 requiere reescritura de URLs en el servidor.

Ventajas de las SPAs

⚡
Navegación rápida
Sin recargas de página: solo cambia el componente necesario. Experiencia fluida tipo app nativa.
📦
Separación frontend/backend
El backend solo expone una API REST/GraphQL. El frontend consume datos y renderiza independientemente.
🔄
Menor carga en servidor
El servidor no genera HTML en cada petición. Solo sirve datos JSON, reduciendo el procesamiento.
🧩
Reutilización de componentes
Los componentes Vue se reutilizan entre vistas. Un Navbar o Button definido una vez se usa en toda la app.
📱
PWA ready
Las SPAs se convierten fácilmente en Progressive Web Apps con Vite PWA plugin y Service Workers.
🛠️
Dev Experience superior
Hot Module Replacement (HMR): los cambios en código se reflejan al instante sin perder el estado.
router/index.js — Configuración Vue Router 
import { createRouter, createWebHistory } from 'vue-router'

// Lazy loading: los componentes se importan solo cuando se necesitan
// Esto mejora el tiempo de carga inicial (code splitting)
const Inicio  = () => import('@/views/Inicio.vue')
const Perfil  = () => import('@/views/Perfil.vue')
const Config  = () => import('@/views/Config.vue')

// Definición de rutas: cada ruta mapea un path a un componente
const routes = [
  { path: '/',       component: Inicio,  name: 'inicio' },
  { path: '/perfil', component: Perfil,  name: 'perfil' },
  { path: '/config', component: Config,  name: 'config',
    // meta: información extra para guards de navegación
    meta: { requiresAuth: true }
  },
  // Ruta con parámetro dinámico: /user/42, /user/juan
  { path: '/user/:id', component: () => import('@/views/User.vue') },
  // Catch-all: cualquier ruta no definida → 404
  { path: '/:pathMatch(.*)*', component: () => import('@/views/404.vue') }
]

const router = createRouter({
  history: createWebHistory(), // usa History API (sin #)
  routes
})

// Navigation Guard: se ejecuta ANTES de cada cambio de ruta
// Permite proteger rutas que requieren autenticación
router.beforeEach((to, from, next) => {
  if (to.meta.requiresAuth && !isAuthenticated()) {
    next({ name: 'login' })   // redirige si no está autenticado
  } else {
    next()                     // permite la navegación
  }
})

export default router
05 — Ciclo de Vida

Montaje, Actualización & Desactivación

Vue expone lifecycle hooks que permiten ejecutar código en momentos específicos: antes/después de crear, montar, actualizar o destruir un componente.

🟢 Creación & Montaje

Creación
setup() / onBeforeMount
setup() es el punto de entrada de la Composition API. Se ejecuta antes del montaje. Los datos reactivos, computed y watchers se declaran aquí. El DOM aún NO existe en esta fase.
Montaje
onMounted()
El componente se insertó en el DOM. Aquí puedes acceder al DOM real (el.$refs), hacer peticiones fetch() iniciales, inicializar librerías de terceros (charts, mapas, etc).

🔄 Actualización & Desactivación

Actualización
onBeforeUpdate / onUpdated()
Se disparan cuando datos reactivos cambian y el DOM se re-renderiza. onBeforeUpdate: DOM aún con valores anteriores. onUpdated: DOM ya refleja el nuevo estado.
Activación
onActivated / onDeactivated()
Solo aplica a componentes dentro de <KeepAlive>. onActivated: el componente se muestra de nuevo (no se recreó). onDeactivated: se oculta pero se mantiene en memoria.
Desmontaje
onBeforeUnmount / onUnmounted()
El componente se elimina del DOM. Aquí debes limpiar: cancelar timers (clearInterval), remover event listeners, cancelar peticiones en curso, desconectar WebSockets.
CicloDeVida.vue — Todos los hooks 
<script setup>
import {
  ref, onBeforeMount, onMounted,
  onBeforeUpdate, onUpdated,
  onBeforeUnmount, onUnmounted,
  onActivated, onDeactivated,
  onErrorCaptured
} from 'vue'

const datos = ref(null)
let   timer = null

// CREACIÓN: setup() se ejecuta primero, DOM no existe aún
onBeforeMount(() => {
  console.log('1. Antes del montaje — DOM no disponible')
})

// MONTAJE: DOM listo → ideal para fetch y librerías externas
onMounted(async () => {
  console.log('2. Componente montado — DOM disponible ✅')
  datos.value = await fetch('/api/datos').then(r => r.json())
  // Iniciar timer (debe limpiarse en onUnmounted)
  timer = setInterval(() => actualizarDatos(), 5000)
})

// ACTUALIZACIÓN: datos reactivos cambiaron, DOM se re-renderizará
onBeforeUpdate(() => {
  console.log('3. Antes de actualizar — DOM aún con valores viejos')
})

// ACTUALIZACIÓN COMPLETA: DOM sincronizado con el nuevo estado
onUpdated(() => {
  console.log('4. DOM actualizado — refleja el nuevo estado ✅')
})

// ACTIVACIÓN (KeepAlive): componente visible de nuevo sin recrearse
onActivated(() => {
  console.log('5. Componente activado (KeepAlive) — estado preservado')
})

// DESACTIVACIÓN (KeepAlive): componente oculto, pero en memoria
onDeactivated(() => {
  console.log('6. Componente desactivado — pausar procesos costosos')
})

// DESMONTAJE: limpiar recursos para evitar memory leaks
onBeforeUnmount(() => {
  clearInterval(timer)  // ⚠️ siempre limpiar timers
})

onUnmounted(() => {
  console.log('7. Componente desmontado — memoria liberada 🧹')
})

// CAPTURA DE ERRORES: atrapa errores de componentes hijos
onErrorCaptured((error, instance, info) => {
  console.error('Error capturado:', error, info)
  return false  // false = no propagar el error hacia arriba
})
</script>
